### Isntsta
안녕하세요, 이걸 읽고 있다는 본 프로젝트(웹사이트)에 꽤 관심을 가지고 있다는 뜻이겠군요.
이미 아시겠지만 이건 Instagram에서 날 팔로우해놓고 정작 내가 팔로우하면 나중에 나를 언팔하는(정작 나를 팔로우 목록에서 같이 지우지는 않는) 사람들때문에 만들었습니다.
그런데 사실 코드를 다 만들어놓고 보니깐 이미 이런걸 해주는 사이트들이 있더군요? 이런 사이트들과 경쟁이 될 지는 잘 모르겠습니다만 뭐 굳이 경쟁 목적으로 만든건 아니니깐 상관은 없겠죠..

프로젝트 이름을 뭘로 할까 고민하다가, 이 기능의 주된 관심사가 'Unfollower'잖아요? 여기서 Un이 '부정' 접두사니깐 이걸 Insta에 넣어서 I**s**nsta라고 해봤습니다. 근데 되게 입에 감기지가 않아서 나중에는 이 이름을 다른걸로 바꿀 것 같아요. 많은 추천 부탁드립니다.

------*------*------

메인 코드 파일은 app.py입니다. 앱 내 전반적인 로직과 UI를 담당합니다. dataParser.py와 analyzer.py는 이 웹사이트의 실질적인 기능을 담당합니다. app.py에서 "맞팔 분석 시작" 버튼을 누르면 dataParser과 analyzer의 각 라이브러리를 호출하는 식이죠. 객체 지향 프로그래밍으로 만들어보려고 일부러 이렇게 파일을 나누어봤습니다만, 코드 라인 자체가 짧아서 절차 지향식으로 프로그래밍했어도 크게 문제될 부분은 없어 보이긴 합니다. 아무튼 그리고 requirements.txt는 python에 내장되어 있는 기본 라이브러리 말고 추가로 불러와야 할 라이브러리들을 담고 있습니다. step1-3 사진들은 웹사이트 중 '사용 방법'에 있는 사진들을 담당합니다.
